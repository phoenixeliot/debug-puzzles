Puzzle Part 1. The Call Stack

Open the Sources panel in DevTools to find the secret password!

For this page, use the Call Stack section. Note that the call stack in the debugger is "upside down" if compared to the way people usually refer to call stack directions. Going "up" the call stack means going towards the first function that was called and away from the most recent. In the debugger, the first function is listed at the bottom, so going "up" means going further down the list shown there.

Usually, we care about seeing the bottom of the call stack (most recently called function) first. If the direction you read somewhere is confusing, you can infer from context what they mean.

Tips & Tricks

  1. To open DevTools, you can right-click on the page and choose the "Inspect" option, or use the keyboard shortcut.

    • Mac: Cmd-Option-i
    • Windows/Linux: Ctrl-Shift-i

    Then refresh the page with the debugger open.

  2. If the debugger pauses without showing you the correct code, click the first entry in the Call Stack to jump to where it's really paused. This usually only happens the first time we pause on a given page.

  3. If it pauses on code that is obviously not part of the puzzle, this is probably due to chrome extensions. Just hit play, or open the page in an incognito window.
